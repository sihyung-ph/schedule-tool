📋 라이브 테이블 복사

타임라인 한 줄(밴드)이 그룹 하나. 첫 줄 = 시작번호, 다음 줄 +1.
각 그룹 안 첫 항목의 index 번호. (예: 1 → _1,_2… / 0 → _0,_1…)
비우면 전체. 지정 시 시작일이 이 날짜 이후(포함)인 스케쥴만.
비우면 끝까지. 지정 시 시작일이 이 날짜 이전(포함)인 스케쥴만.
켜면 「이 날짜까지」 무시하고 타임라인 끝날(시작일+기간−1)까지 사용.

📋 토너먼트 추출

같은 토너 DefKey 막대들 → 1개 컨텐츠로 묶임. ContentId = 토너 DefKey 그대로, 각 막대 = CustomSeasonData(시즌). DefKey 는 📤 추출 설정에서 입력.
비우면 전체. 지정 시 시작일이 이 날짜 이후(포함)인 막대만 시즌에 포함.
⚙ 고정값 설정 (모든 스케쥴 토너 공통)
카테고리 ID CategoryIds
토너 그룹 매칭 시 내려올 플레이데이터 키. 쉼표로 여러개. (스케쥴 토너 = Account)
채널 SeparateByUserTags
매칭그룹 필터를 서버 채널(ChannelId)로 사용. 0 = ChannelId 0 (기본 채널). 쉼표로 여러개.
최대 그룹 크기 MaxEntryGroupSize
토너 그룹 하나에 들어갈 최대 인원 수.
전 시즌 보상 개수 RewardClaimableSeasonCount
보상 받을 수 있는 직전 시즌 개수. (약속값 = 1)
보상 수령 보장(분) RewardDurationMin
직전 시즌 종료 후 이 시간까지 보상 수령 보장. 10080분 = 7일.
게임 타이틀 GameTitle
PK 생성용. PK = GP)CONTENT#<GameTitle>.
점수 갱신 타입 SetScoreType
0 = Increase (점수 누적 증가식). 우리는 Increase만 사용.
순환 주기 PeriodUnit
스케쥴 토너는 항상 Custom(6). PeriodValue = 막대 기간(days), 시즌은 CustomSeasonDatas 로 입력됨.
고정 베이스 JSON 고정 클랜 토너먼트
운영툴에 항상 들어가야 하는 고정 클랜 토너먼트 레코드(ClanQuest · ClanRaidStage 등). 운영툴 [토너먼트 > 데이터 추출/덮어쓰기]에서 JSON 추출 → 여기 붙여넣기. 매 추출 시 이 베이스가 스케쥴 토너 앞에 그대로 포함됨. 베이스가 바뀌면 다시 붙여넣기. (비워두면 스케쥴 토너만 추출)
